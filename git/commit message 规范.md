
## Commit message 的作用

* 提供历史信息，方便浏览
* 过滤不重要信息，便于查找
* 可以直接使用commit生成Change log


## 格式
每次提交，Commit message 都包括三个部分：Header，Body 和 Footer。

**其中，Header 是必需的，Body 和 Footer 可以省略。**


### Header
Header部分只有一行，包括三个字段：`type`（必需）、`scope`（可选）和`subject`（必需）。

#### type
* feat:新功能，feature
* fix:修补bug
* chore:改变构建流程或依赖库、辅助工具……
* test:测试用例
* style:仅代码样式，不改逻辑
* refactor：仅重构代码
* revert:回退版本
